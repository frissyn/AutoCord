Rules

These are things that any bot must do in order to be added to the Repl.it Discord server.

  •  A bot must follow all the server's rules. If a member isn't allowed to do something, a bot isn't either. This includes (but is not limited to) no bad language, no spamming, and no inappropriate content.
  •  A bot should never send any message unless explicitly triggered by a command. This means any features such as deleted message logging or level-up messages should be disabled for the Repl.it server.
  •  A bot must be open source. This is to ensure that it is not malicious.

Guidelines

These are things that are strongly recommended for bots to do, but they don't have to.

  •  A bot should have a very obvious point of entry. The best way to do this is to have a help command in its status.
  •  A bot should not ping users unless absolutely necessary.
  •  A bot should have a customisable prefix to ensure that no two bots have the same prefix. The ideal solution is for a bot's prefix to be its @mention.
